Twitter

Twitter is the best way to keep up on the industry. Quick tips, links to articles, demos, and events - there's really no better place to get information.

  • Smashing Magazine - Great articles on development, design, UX, and all things in between.
  • CSS-Tricks - Technical demos and tutorials, covering more than just CSS.
  • Open Web Platform Daily Digest - Offers a round-up of the latest web development news. Very handy for staying on top of things (even if you just read the headlines).
  • Front-end Rescue - Quality links and re-tweets.
  • A List Apart - A seminal entity in the world of web standards. The original producer of high-quality web development articles.
  • JavaScript Daily - Lots of JavaScript related links.
  • Firefox DevTools - Helpful tips for using the Firefox Dev Tools and some good links and re-tweets.
  • Chrome DevTools - Just like Firefox DevTools, but for Chrome.
  • Paul Irish - Works on developer productivity for Chrome. Performance tips, articles, and other good links.
  • Christian Heilmann - Developer outreach for Microsoft, former evangelist for Mozilla. Open web enthusiast. Quality development related commentary & links, pictures of hedgehogs.

There are many other great accounts to follow. If you're looking for more, just check the following list of any account above.

Text Editors

There is no shortage of options when it comes to text editors. I'll limit this list to the three most popular that are available for all major platforms (Mac, Windows, & Linux).

  1. Sublime Text (currently on version 3) is by far the most popular text editor. It's clean, simple, and has a huge community and plugin library. Free to try (at the moment), $70 for a license.
  2. Atom is rather new on the scene, and is very comparable to Sublime Text. Free!
  3. Brackets is another up and coming text editor that provides a bit more integrated features than Sublime Text and Atom. Free!

Testing in IE

As a developer, IE is probably not your browser of choice. However, usage rate is still very high. Any website you ship must be tested in IE.

If you're not on a Windows machine, testing old(er) versions of IE is going to be a little inconvenient. You'll need to set up some virtualization software and actually install multiple versions of Windows. VMWare, Parallels, and VirtualBox are all good virtualization options.

Next, head to modern.IE and download all the virtual machines you think you'll need. It's pretty safe to start with IE 9, 10, & 11. Depending on the project/client, you may need to grab IE 8 as well.

In addition to virtual machines, Microsoft offers a remote testing service that's a bit less involved than installing virtual machines. As of this writing, the remote service only offers IE 11 Technical Preview. Hopefully they'll offer additional IE versions through this service in the near future.

Other Browsers

Safari is rather widely used and should be tested (if you have access to OS X). Opera is not terribly popular. I almost never test in it, and I've never had a client or user report Opera bugs. Opera uses the Blink rendering engine, developed by Google and used in Chrome. If your site works in Chrome, it's safe enough (these days) to assume it works in Opera as well.

Touch Browsers

Browsers on phones and tablets obviously cannot be ignored. They are generally too numerous and varied to realistically test on all. You can usually get away with testing only one of the major platforms: Android (Chrome & Firefox) and iOS (Safari & Chrome). If you can afford it, get a phone and a tablet to test your responsive breakpoints. If not, you can get by with just a phone.

Version Control

What is version control, anyway? In short, it's a way to manage changes to your project's source code. The concepts can be a little tricky, but most development teams (and even individuals) find it indispensable. The primary benefits of version control are:

  • Tracking changes to each file, meaning if you mess something up, you can easily go back to the last working version.
  • Ability for multiple developers to work in tandem and (often painlessly) merge their changes.
  • Easily revert code changes that introduce bugs or errors.

The most popular version control software today is Git. Subversion, while not as popular as it once was, is still rather widely used as well.

Most teams put their Git repositories (which hold all project code) online for easy sharing and viewing. GitHub is by far the most popular Git host, but Bitbucket is also highly regarded.

It may not make sense to dive in to the world of version control right away. However, you should create a GitHub account straight away. (Many companies will ask for your GitHub URL during the application process.)
